데이터베이스는 관련성을 가지며 중복이 없는 데이터들의 집합이다. 이러한 데이터베이스를 관리하는 시스템을 DBMS(DataBase Management System)데이터베이스 관리 시스템이라고 한다. 
보통 서버의 하드 디스크나 SSD 등의 저장 매체에 데이터를 저장합니다. 저장 매체가 고장나거나 사용자가 직접 데이터를 지우지 않는 이상 계속 데이터가 보존되므로 
서버 종료 여부와 상관없이 데이터를 지속적으로 사용할 수 있다, 

또한, 서버에 데이터 베이스를 올리면 여러 사람이 동시에 사용할 수 있다. 사람들에게 각각 다른 권한을 줘서 어떤 사람을 읽기만 가능하고, 어떤 살마은 모든 작업을 
가능하게 할 수 있다. 

데이터베이스를 관리하는 DBMS는 RDMS라고 부르는 관계형 DBMS가 많이 사용됩니다. 대표적인 RDBMS로는 Oracle, Mysql 등이 있다.

콘솔로는 데이터를 한눈에 보기에 무리가 있는데, 워크벤치라는 프로그램은 데이텁 베이스 내부에 저장된 데이터를 시각적으로 관리할 수 있음. 
sql 쿼리문 뒤에는 세미콜론(;)을 붙혀야 실행된다

테이블 생성하기 
INT 는 정수를 의미한다. 소수까지 저장하고 싶다면 FLOAT이나 DOUBLE 이나 자료형을 사용하면 된다. 
VARCHAR(자릿수) 이오ㅔ도 CHAR(자릿수)라는 자료형도 있따, CHAR 는 고정 길이고, VARCHAR는 가변 길이이다. 예를 들어 
CHAR(10)이면 반드시 길이가 10인 문자열만 넣어야 하고, VARCHAR(10) 일 경우에는 길이가 0~10인 문자열을 넣을 수있따. 
TEXT: 는 긴글을 저장할때 사용 
TINYINT: 는 -128 부터 127까지의 정수를 저장할 때 사용한다. 1또는 0만 저장한다면 불(boolean) 값 과 같은 역할을 할 수 있음. 

자료형 뒤에 NOT NULL, NULL, UNSINGED, AUTO_INCREMENT, DEFAULT 등의 옵션이 붙어 있음. 
NULL 과 NOT NULL은 빈칸 허용할지 묻는 옵션 
AUTO_INCREMENT 숫자를 저절로 올리겠다는 의미,
UNSINGED는 숫자 자료형에 적용 되는 옵션임, 숫자 자료형은 기본적으로 음수 범위를 지원한다. 만약에 이게 적용되면 0~4294967295까지 저장할 수 있따. FLOAT 나 DOUBLE에는 적용 불가능. 
PRIMARY KEY(id) 는  고유한 번호 id에 제공한다는 뜻이다,
UNIQUE INDEX는 해당 값이 고유해야 하는지에 대한 옵션이고, name 컬럼이 해당된다. 인덱스 이름은 name_UNIQUE로 nma 컬럼을 오름차순으 ASC 내림차순 DESC

COMMENT는 테이블에 대한 보충 설명, 
DEFAULT CHARCTER SET을 utf8로 설정 안하면 한글 입력 X
ENGINE은 여러가지가 있는데, InnoDB가 제일 많이 사용된다.
